# ОСНОВНОЙ КОНТЕКСТ ПРОЕКТА:
Я разрабатываю веб-приложение под названием "Interview Master AI" (или любое другое рабочее название). Это универсальная платформа для подготовки к собеседованиям, предназначенная для широкого круга пользователей, особенно для начинающих специалистов, которые испытывают неуверенность и страх перед реальными интервью.
## Ключевые особенности проекта:
- ИИ-генерируемые вопросы: Система использует искусственный интеллект для динамической генерации вопросов, имитирующих реальное собеседование.
- Экспертное управление ИИ: ИИ не действует автономно. Специалисты-эксперты (HR-менеджеры, отраслевые специалисты) могут создавать и настраивать "курсы" или "сценарии", определяя:
	Какие вопросы считать важными.
        Какие ответы являются "хорошими" (желательными).
        Какие ответы являются "плохими" (нежелательными/ошибочными).
        Контекст и сложность вопросов.
- Анализ ответов пользователя: ИИ анализирует текстовые ответы пользователя в реальном времени (через чат-диалог) на основе экспертных настроек, предоставляя обратную связь.
- Геймификация: Для повышения мотивации и вовлеченности пользователей внедрены элементы геймификации:
        "Стрики" (Streaks): Отслеживание и поощрение регулярности занятий.
        "Достижения" (Achievements): Награды за выполнение определенных задач или достижение этапов (например, "Пройдено 10 собеседований", "Отвечено на 50 вопросов").
- Интерактивный чат-диалог: Основное взаимодействие пользователя с ИИ происходит в формате текстового чата.
- Универсальность: Платформа должна быть адаптируемой для подготовки к собеседованиям в различных областях (IT, маркетинг, финансы, HR, и т.д.) за счет гибких экспертных настроек.
### Цель 
Создать эффективный, интерактивный и мотивирующий инструмент, который поможет пользователям преодолеть страх перед собеседованиями, улучшить коммуникативные навыки и получить ценный опыт в безопасной среде.
### Технологическая база
Next.js fullstack + внешние API для ИИ. POSTGRESQL
# РЕЗУЛЬТАТЫ ЭТАПА «АНАЛИЗ ПРЕДМЕТНОЙ ОБЛАСТИ» (СДЕЛАНО):
1. Акторы системы:
Администратор: Управляет категориями (отраслями), глобальными достижениями, экспертами.
Эксперт (HR): Создает сценарии в рамках категорий, настраивает критерии (стоп-слова, фокусы оценки).
Соискатель (Пользователь): Проходит интервью, копит XP, повышает уровень (Стажер -> Техлид), держит стрики.

## Архитектура
1. Особенности реализации системы
Для разработки веб-платформы «Interview Master AI», направленной на подготовку пользователей к собеседованиям с использованием искусственного интеллекта, мы будем руководствоваться следующими принципами и подходами:
Проектирование пользовательского интерфейса (UI/UX):
•	Создание адаптивного и минималистичного интерфейса на базе библиотеки Shadcn/UI и Tailwind CSS для обеспечения высокой скорости работы и концентрации пользователя на диалоге.
•	Реализация интерактивного чат-интерфейса, имитирующего реальную среду мессенджеров или платформ для видеосвязи.
•	Управление ИИ-логикой и контекстом:
•	Использование библиотеки LangChain для управления жизненным циклом диалога и «памятью» нейросети.
•	Приоритетное использование системных промптов (System Prompts) для передачи экспертных настроек сценария, что минимизирует риск «забывчивости» ИИ в процессе длительных сессий.
•	Реализация пошагового анализа ответов пользователя в реальном времени с последующей агрегацией данных в итоговый отчет.
Геймификация и вовлечение:
•	Разработка системы «стриков» и прогрессии опыта (XP), работающей в реальном времени.
•	Использование транзакционных механизмов на уровне базы данных для обеспечения целостности данных при одновременном обновлении прогресса пользователя и завершении сессии.
Управление медиа-контентом и кэшированием:
•	Интеграция с S3-совместимым хранилищем (например, Cloudflare R2 или Tigris) для надежного хранения пользовательских аватаров и иконок достижений.
•	Использование Redis для кэширования ответов и хранения временных состояний сессий, что снижает нагрузку на основную БД и уменьшает задержки (latency) при взаимодействии.
Безопасность:
•	Централизованное управление доступом через BetterAuth.
•	Изоляция API-ключей и секретов на стороне сервера с полным исключением их передачи на клиентскую часть.
