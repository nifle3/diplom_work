# Диаграммы последовательностей

Этот раздел собран по одному правилу:
1. Сначала фиксируем кейсы.
2. Затем перечисляем участников каждого кейса.
3. После этого описываем сам поток в `mermaid` sequence diagram.

Я сознательно не дублирую чистые утилиты и вычислительные функции, где нет обмена между отдельными участниками. Их поведение уже отражено в `docs/tests`.

## Общие Участники

- `User` или `Expert` или `Admin` - внешний инициатор.
- `Web UI` - страница или форма в `apps/web`.
- `tRPC API` - роутер из `packages/api`.
- `Postgres` - хранилище, доступ через Drizzle.
- `LLM` - сервис для суммаризации, оценки и планирования вопросов.
- `Logger` - запись событий и ошибок.
- `File storage` - только там, где нужны ссылки на изображения.
