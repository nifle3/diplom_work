# 4.7.2.8 Математическое обеспечение

Математическое обеспечение системы включает математические модели и методы, реализованные для автоматизированной обработки данных в рамках AI-платформы проведения собеседований. Ниже описаны все ключевые математические компоненты.

---

## 1. Формульный движок достижений

**Файл:** `packages/api/src/achievements/formula.ts`

### 1.1 Формальная грамматика выражений

Для описания условий выдачи достижений используется мини-язык булевых и арифметических выражений. Грамматика (в нотации EBNF):

```
expression  = or_expr
or_expr     = and_expr  { "||" and_expr }
and_expr    = eq_expr   { "&&" eq_expr }
eq_expr     = cmp_expr  { ("==" | "!=") cmp_expr }
cmp_expr    = add_expr  { (">" | ">=" | "<" | "<=") add_expr }
add_expr    = mul_expr  { ("+" | "-") mul_expr }
mul_expr    = unary     { ("*" | "/" | "%") unary }
unary       = ("!" | "-") unary | primary
primary     = NUMBER | IDENTIFIER | "(" expression ")"
```

Грамматика определяет 7 уровней приоритета операторов, что соответствует стандартной математической иерархии.

### 1.2 Лексический анализ (токенизация)

Метод `tokenize(formula: string): Token[]` выполняет линейный проход по строке формулы и преобразует её в последовательность токенов. Сложность: **O(n)**, где n — длина строки.

Типы токенов:
- `number` — числовой литерал (целое или вещественное)
- `identifier` — переменная контекста
- `operator` — бинарный или унарный оператор
- `paren` — скобка `(` или `)`

### 1.3 Синтаксический анализ (парсинг)

Класс `FormulaParser` реализует метод рекурсивного спуска (Recursive Descent Parsing). Каждому уровню приоритета соответствует отдельный метод:

| Метод               | Оператор(ы)            | Приоритет |
|---------------------|------------------------|-----------|
| `parseOr()`         | `\|\|`                 | 1 (низший)|
| `parseAnd()`        | `&&`                   | 2         |
| `parseEquality()`   | `==`, `!=`             | 3         |
| `parseComparison()` | `>`, `>=`, `<`, `<=`   | 4         |
| `parseAdditive()`   | `+`, `-`               | 5         |
| `parseMultiplicative()` | `*`, `/`, `%`      | 6         |
| `parseUnary()`      | `!`, унарный `-`       | 7 (высший)|

Результат парсинга — абстрактное синтаксическое дерево (AST) типа `FormulaNode`.

### 1.4 Вычисление AST

Функция `evaluateNode(node, context)` выполняет рекурсивный обход дерева:

- Листовые узлы: возвращают числовое значение переменной из контекста или числовой литерал.
- Унарные узлы: применяют `!` (логическое НЕ) или `-` (арифметическое отрицание).
- Бинарные узлы: вычисляют оба операнда и применяют оператор.

Итоговый результат приводится к типу `boolean` функцией `Boolean(...)`.

**Пример формулы достижения:**
```
xp >= 1000 && streak >= 7 && completedInterviews >= 10
```

## 2. Вычисление метрик пользователя

**Файл:** `packages/api/src/achievements/metrics.ts`

### 2.1 Нормализация дат по UTC

Для корректного сравнения дат все временные метки нормализуются к началу суток по UTC:

```
getUtcDayStart(date) = date с установкой часов, минут, секунд и миллисекунд в 0 по UTC
```

### 2.2 Вычисление количества прошедших дней

```
calculateDaysSince(date, now) = max(0, floor((getUtcDayStart(now) − getUtcDayStart(date)) / 86 400 000))
```

где 86 400 000 мс = 24 × 60 × 60 × 1000 — количество миллисекунд в сутках.

### 2.3 Алгоритм подсчёта серии (streak)

**Функция:** `calculateStreakFromSessions(sessions)`

**Входные данные:** массив сессий со статусами и временными метками.

**Алгоритм:**

1. Отфильтровать завершённые сессии (`status = complete`).
2. Преобразовать временные метки в ключи дат формата `YYYY-MM-DD` по UTC.
3. Дедуплицировать ключи (множество уникальных дней).
4. Отсортировать в убывающем порядке.
5. Если последний день — не сегодня и не вчера → вернуть `0`.
6. Итерировать по дням назад, подсчитывая непрерывную последовательность:

```
streak = 0
expectedDay = completedDays[0]
for each dayString in completedDays:
    if dayString ≠ expectedDay: break
    streak++
    expectedDay = dayString − 1 день
return streak
```

### 2.4 Агрегация статистики сессий

Из отфильтрованных завершённых сессий вычисляются следующие метрики:

| Метрика           | Формула                                              |
|-------------------|------------------------------------------------------|
| `averageScore`    | `Σ(scores) / len(scores)` (0 если нет завершённых)  |
| `bestScore`       | `max(scores)` (0 если нет завершённых)              |
| `lastScore`       | `scores[0]` — последняя завершённая (0 если нет)    |
| `interviewsToday` | `count(sessions where startedAt.date == today)`     |
| `completedToday`  | `count(completed where completedAt.date == today)`  |

### 2.5 Алгоритм синхронизации достижений

**Функция:** `syncUserAchievements(client, userId)`

Реализует цикл с фиксированной точкой для обработки каскадных достижений:

```
while true:
    context = getAchievementContext()       // пересчёт всех 13 метрик
    existingIds = getExistingUserAchievementIds()
    newAwards = []
    for each achievement not in existingIds:
        if evaluateAchievementFormula(achievement.formula, context):
            newAwards.push(achievement)
    if newAwards is empty: break
    insert newAwards into DB
```

Цикл завершается, когда ни одно новое достижение не выдаётся. Это гарантирует корректную обработку достижений, которые зависят от значения `achievementCount` (счётчик самих достижений).

## 3. Вычисление опыта (XP) за сессию

**Файл:** `packages/api/src/routers/sessionExperience.ts`

```
XP(score) = max(0, round(score))
```

- **Тип функции:** кусочно-линейная с нижней границей 0.
- **Область значений:** `[0, 100]` (score ограничен диапазоном 0–100).
- **Округление:** `Math.round()` — стандартное арифметическое округление.
- При `score = null` возвращается 0 (отменённые сессии не дают XP).

Таким образом, накопленный XP пользователя равен сумме по всем завершённым сессиям:

```
totalXP = Σ XP(scoreᵢ)  для всех завершённых сессий i
```

## 4. Шкала интерпретации оценки

**Файл:** `apps/web/src/lib/score.ts`

Числовая оценка сессии (0–100) отображается в качественную категорию по пороговой функции:

| Диапазон    | Категория          | Отображение          |
|-------------|--------------------|----------------------|
| [80, 100]   | Сильный результат  | Зелёный              |
| [60, 79]    | Хорошая база       | Жёлтый               |
| [40, 59]    | Есть зоны роста    | Оранжевый            |
| [0, 39]     | Есть зоны роста    | Красный              |

Формально:

```
category(score) =
    "Сильный результат",  если score ≥ 80
    "Хорошая база",       если 60 ≤ score < 80
    "Есть зоны роста",    если score < 60
```

## 5. Структурированный вывод LLM (схемы оценивания)

**Файл:** `packages/llm/src/evaluater/evaluateAnswer.ts`

### 5.1 Схема оценки ответа

LLM возвращает структурированный ответ, валидируемый Zod-схемой:

```typescript
outputSchema = z.object({
    score:        z.number().int().min(0).max(100),
    feedback:     z.string().min(1),
    analysisNote: z.string().min(1),
    strengths:    z.array(z.string()).max(3),
    improvements: z.array(z.string()).max(3),
})
```

- **score** — целочисленная оценка ответа в диапазоне [0, 100].
- **strengths / improvements** — не более 3 элементов каждый (ограничение модели).

Оценка может работать в двух режимах:
- **`answer`** — оценка одного ответа на конкретный вопрос.
- **`session`** — оценка всей беседы (массив пар `{question, answer}`).

### 5.2 Схема планирования шага интервью

**Файл:** `packages/llm/src/interviewer/planInterviewStep.ts`

```typescript
outputSchema = z.object({
    decision: z.enum(["follow_up", "next_topic", "finish"]),
    question: z.string()
})
```

Конечный автомат решений LLM:

```
follow_up  → задать уточняющий вопрос по текущей теме
next_topic → перейти к следующей теме скрипта
finish     → завершить интервью (все темы раскрыты)
```

## 6. Алгоритм инкрементального суммаризирования

**Файл:** `packages/llm/src/summarization/summarize.ts`

Для сжатия контекста диалога применяется метод скользящего суммаризирования:

```
summary₀ = ""
for i = 1..N:
    summaryᵢ = LLM.summarize(question_i, answer_i, summary_{i-1})
return summaryₙ
```

Метод позволяет поддерживать контекст произвольно длинного диалога в пределах фиксированного контекстного окна языковой модели.

## 7. Постраничная навигация

**Файл:** `packages/api/src/routers/script.ts`

```
offset = (page − 1) × limit
pages  = ⌈total ÷ limit⌉
```

- **Смещение выборки:** `(page - 1) × limit` — стандартная формула пагинации.
- **Число страниц:** потолочное деление `Math.ceil(total / limit)`.
- **Ограничения:** `limit ∈ [1, 50]`, по умолчанию 12.

## 8. Алгоритм ограничения частоты запросов (Rate Limiting)

**Файл:** `packages/ratelimit/src/index.ts`

Применяется метод скользящего окна (Sliding Window):

```
Condition(t): count(requests in (t − window, t]) ≤ quota
```

Настроенные лимиты:

| Лимитер   | Окно    | Квота              | Назначение             |
|-----------|---------|--------------------|------------------------|
| `global`  | 10 с    | 20 запросов        | Все эндпоинты          |
| `llm`     | 1 мин   | 5 запросов         | Вызовы AI-оценивания   |
| `email`   | 1 час   | 3 письма           | Email-уведомления      |
| `page`    | 1 мин   | 30 запросов        | Страничная навигация   |

Идентификатор клиента определяется по приоритету: `userId → IP → UserAgent → "anonymous"`.

## 9. Нормализация диалога

**Файл:** `packages/api/src/routers/session.ts`

Алгоритм восстановления пар «вопрос–ответ» из линейного потока сообщений чата:

```
pairs = []
for each message in messages:
    if message.isAi:
        pairs.push({ question: message.text, answer: "" })
    else:
        if pairs[-1].answer == "":
            pairs[-1].answer = message.text
        else:
            pairs.push({ question: "", answer: message.text })
return pairs.filter(p => p.question ≠ "" && p.answer ≠ "")
```

Результат передаётся в LLM для оценки завершённой сессии.

## 10. Алгоритм ранжирования последних активностей

**Файл:** `packages/api/src/routers/activity.ts`

```
activities = sessions
    .filter(s => s.status ∈ {complete, canceled})
    .sortBy(s => s.date, descending)
    .slice(0, 3)
```

Сложность: **O(n log n)** для сортировки + **O(1)** для среза.
